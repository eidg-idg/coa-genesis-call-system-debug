Project Overview

What Is This Project?

When a phone call comes in through Genesys Cloud, Salesforce automatically "screen pops" a verification page so the CSR can confirm the caller's identity. This project ensures that workflow is reliable, captures all required data, and matches the experience CSRs see when they use the EPlus manual verification process.

Why Does It Matter?

  • Verification records were intermittently not saved, creating compliance gaps
  • Caller Name and Phone were hidden for most call types
  • The Genesys screen pop and EPlus manual flows show different fields in a different order, confusing CSRs
  • Achieving parity reduces training burden and improves audit readiness

Two Bodies of Work

Body 1 — Remediation Code Complete
Fix the save-before-close race condition, make Caller Name/Phone always visible, pass ANI end-to-end, add error handling.

Body 2 — Conditional Rendering Parity Active
Reverse-engineer the EPlus managed package UI and align the Genesys screen pop so both paths show identical fields, validations, and conditional logic.

Key Decisions

Save-Before-Close

Verification records are now saved before the modal closes. If the save fails, the modal stays open with an error message and the CSR can retry.

Caller Identity Always Captured

Caller Name is required on both Member and Provider paths. Caller Phone is optional and auto-filled from the Genesys ANI when available.

Parity by Observation

Because COA does not own the EPlus source code, all parity decisions are based on observed runtime behavior, not assumptions about internal logic.

Phased Deployment

The trigger handler refactor deploys separately from the call-flow fixes to reduce risk. Both bodies of work will be combined for the final production release.
